# 🚀 Ashura Core (ESP-IDF Migration)

This repository represents the **migration of the original _Ashura Core_ project to ESP-IDF**.

The goal of this migration is to move from a higher-level or mixed framework setup to a **fully native ESP-IDF-based architecture**, enabling better performance, deeper hardware control, and improved scalability for advanced embedded use cases.

---

## 🔄 Why Migration?

The original Ashura Core was built with a more abstracted environment.
This version transitions to **ESP-IDF** to:

- ⚡ Gain **low-level control** over ESP32-S3 hardware
- 🧠 Leverage **PSRAM and memory management** more efficiently
- 🛠️ Use **native ESP-IDF components and APIs**
- 📈 Build a **production-ready, scalable firmware architecture**
- 🔍 Improve **debugging, logging, and system reliability**

---

## 🎯 What’s Changed

- Migrated to **ESP-IDF framework**
- Introduced **sdkconfig-based configuration**
- Added **custom partition table**
- Enabled **PSRAM (SPIRAM) support**
- Improved **logging and watchdog handling**
- Structured project for **modular components**

---

## 🧩 Vision

This migration is not just a rewrite —
it’s a step toward building **Ashura Core as a robust embedded platform** capable of handling:

- Complex UI / display systems
- Audio or real-time processing
- Network-enabled features
- Scalable modular firmware design

---

## 🚧 Status

🛠️ Migration in progress — core systems are being restructured and stabilized.
